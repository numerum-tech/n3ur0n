How it works

1

Run an instance

The desktop app gives you a chat UI and a local backend out of the box. The server binary lets you publish capabilities to peers.

2

Author capabilities

A capability is a cap.toml file. Point it at a backend (Ollama, OpenAI, MCP server, plain HTTP) and it shows up in your registry.

3

Connect to peers

Add a peer endpoint. The planner discovers their capabilities, picks the right one, signs the call, and routes the result back.

4

No central authority

Identity is n3: + Base32(SHA-256(Ed25519 pubkey)). Every message is signed. Anyone can verify; nobody can revoke.

What N3UR0N is not

  • Not a model or an inference engine — bring your own backend.
  • Not a new tool-calling format — capabilities are TOML manifests on disk.
  • Not a registry — peer discovery is gossip-based, not curated.
  • Not vendor-tied — Apache-2.0, no telemetry, no required accounts.
